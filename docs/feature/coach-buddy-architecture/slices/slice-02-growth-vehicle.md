# Slice 02: Growth Vehicle (Framework Discovery)

**Goal**: A coach working through a situation can encounter and explore a relevant framework without disrupting their situation-focus flow.

## IN Scope
- Interest detection criteria defined and implemented in SKILL.md
- Deep-dive available on explicit request or detected interest signal
- Mode management redirects using standard coaching language
- Ask-rather-than-assume pattern when mode is ambiguous

## OUT of Scope
- Framework curriculum or structured learning path
- Cross-session memory of which frameworks the coach has encountered
- Automated framework recommendation without situational grounding

## Learning Hypothesis
Disproves: "surfacing framework discovery during situation-focus conversations will disrupt the coach's thinking flow."
Confirms: With explicit interest signals and coaching-register language, Job 2 can coexist with Job 1 without one undermining the other.

## Acceptance Criteria
- Framework deep-dive available via coach request or explicit interest signal — not proactive
- Mode management redirect uses coaching language ("What do you want to achieve here today?")
- Redirect fires only at natural turn boundaries, not mid-thought
- Tool asks when mode is ambiguous rather than defaulting (D4 in practice)
- Situation-focus wins when stakes are stated as consequential or irreversible (D5 in practice)

## Dependencies
- Slice 01 complete and validated
- Interest detection criteria specified (DESIGN wave — requires resolving D2 underdetermined flag)
- D4 ambiguity threshold specified (DESIGN wave — requires resolving D4 underdetermined flag)
- D5 inference-or-stated resolved (DESIGN wave — requires resolving D5 underdetermined flag)
